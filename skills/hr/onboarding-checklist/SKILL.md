# Onboarding Checklist

## Description
Create and track the onboarding checklist for new employees: prepare equipment, accounts, training, and team introductions.

## Trigger
- A new employee is about to start
- User says "onboarding", "new employee", "new hire", "prepare for new joiner"
- User needs a checklist for the first day of work

## Instructions

### When to activate
- Prepare everything before a new employee starts
- Create a first-week onboarding schedule
- Track completion progress of onboarding items
- Create a welcome kit of information for the new joiner

### Checklist by Phase

**Before the start date (T-7 to T-1)**
- [ ] Prepare the employment contract
- [ ] Request IT to set up laptop/equipment
- [ ] Create company email and accounts (Slack, Jira, etc.)
- [ ] Prepare desk and access card
- [ ] Notify the team about the new joiner
- [ ] Send a welcome email to the new employee

**First Day (Day 1)**
- [ ] Welcome and office tour
- [ ] Hand over equipment, access card, and accounts
- [ ] Introduce the team and assigned buddy/mentor
- [ ] Walk through basic internal processes
- [ ] Welcome lunch

**First Week (Week 1)**
- [ ] Training on company products/services
- [ ] Set up work environment (tools, access)
- [ ] 1:1 meeting with manager to align on goals
- [ ] Read internal documentation (handbook, policy)

**First Month (Month 1)**
- [ ] Complete probation goals
- [ ] Weekly 1:1 check-ins with manager
- [ ] Two-way feedback at end of month

### Rules
- Send the checklist to the manager AND HR at least 7 days in advance
- Customize by department (dev needs GitHub access, marketing needs social accounts)
- Buddy system is mandatory: assign 1 buddy from the same team
- Automatic follow-up: send reminders if items are not completed by the deadline

### Output Format
```
🎉 Onboarding: [New Employee Name]
🎯 Position: [Position Title] - [Department]
📅 Start Date: [DD/MM/YYYY]
👤 Buddy: [Buddy Name]

Progress: [X/Y] items completed ([%])

[Detailed checklist by phase]
```
