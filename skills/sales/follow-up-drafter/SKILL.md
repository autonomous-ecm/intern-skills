# Follow-up Drafter

## Description
Draft follow-up emails/messages after meetings, demos, and proposals. Automatically remind about follow-up schedules and track responses.

## Trigger
- User needs to follow up with a customer
- User says "follow up", "remind", "resend", "check in", "track deal"
- After a meeting/demo, a recap needs to be sent

## Instructions

### When to Activate
- Follow up after the first call/meeting
- Follow up after sending a proposal (day 3, 7, 14)
- Nurture a lead that is not yet ready to buy
- Re-engage a customer who has not responded for a long time
- Send a thank you after closing a deal

### Templates by Stage
- **Post-meeting recap**: Summarize discussion, action items, next steps
- **Post-demo**: Highlight key features shown, answer pending questions
- **Post-proposal (3 days)**: Check if they have reviewed the proposal, offer to clarify
- **Post-proposal (7 days)**: Gentle reminder, add social proof/case study
- **Post-proposal (14 days)**: Last chance, create urgency
- **Win back (90 days)**: Share an update/new feature, re-open conversation
- **Thank you (won deal)**: Confirm next steps, welcome onboard

### Process
1. Identify context: deal stage, which follow-up attempt this is
2. Choose the appropriate template
3. Personalize: add details from the previous conversation
4. Add value: include a useful resource (case study, article, demo recording)
5. Clear CTA: one specific action for the customer

### Rules
- **Rule of 3**: Maximum 3 follow-ups per touchpoint, no spam
- Every follow-up must ADD VALUE, not just "checking in"
- Subject line: reply in the same thread (Re:) to increase open rate
- Timing: send Tuesday through Thursday, 9:00-11:00 AM (highest open rate)
- If no response after 3 follow-ups → move to nurture, do not push

### Output Format
```
📧 Follow-up Email

📌 Context: [Post-meeting / Post-proposal / Re-engage]
🔢 Attempt: [#N of 3]
📅 Scheduled: [Send date and time]

Subject: [Subject line]

---
[Email body]
---

⏰ Next follow-up: [Date] (if no reply)
📊 Deal stage: [Current stage]
```
